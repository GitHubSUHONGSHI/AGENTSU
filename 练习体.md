# 练习题模块规格

## 模块目标

新增一个“练习题”菜单与路由模块，用于给 Python 学习者提供编码练习。页面每次展示 2 道题，顺序固定为先简单题、后难度题。

## 题库规则

- 题库固定为 480 道题。
- 简单题 240 道，面向初学者练习。
- 难度题 240 道，面向进阶练习。
- 题型以编码练习为主，场景尽量贴近实际开发。
- 题库内容不在本文档中完整展开，后续实现时以本文档作为验收依据之一。

## 复合型要求

- 所有题目都不能只考察单一知识点。
- 简单题至少组合 2 个知识点。
- 难度题至少组合 3 个知识点，并且至少跨 2 个章节。
- 小标题或题目提示中必须说明本题涉及到的章节与知识点。

## 题目字段

每道题必须包含以下字段：

- 题目描述：必填，清晰说明开发场景、输入、处理目标与输出要求。
- 涉及章节：列出本题关联的课程章节。
- 涉及知识点：列出本题综合使用的知识点。
- 难度：只能是“简单”或“难度”。
- 答案 Markdown：给出可参考的实现方式。
- 分析 Markdown：说明解题思路、关键步骤、容易出错的位置和改进方向。

## 交互规则

- 刷新题型：每次点击刷新按钮，从题库中更新 2 道题。
- 展示顺序：第 1 道为简单题，第 2 道为难度题。
- 已学会：点击“已学会”后，将该题缓存到本地。
- 排除规则：已学会的题目在后续刷新中不能再次出现。
- 保留概率：未点击“已学会”的题目，下次刷新仍然有概率出现。
- 重置功能：提供重置按钮，清空已学会缓存，将题库恢复到初始化可抽取状态。

## 答案格式

- 答案与分析必须使用 Markdown 格式。
- 代码块统一使用 Python fenced code block。
- Python 代码块格式如下：

```python
def normalize_name(raw_name):
    name = raw_name.strip()
    return name or "未命名用户"
```

## 题型方向

练习题应以实际开发中的编码任务为主，优先覆盖以下场景：

- 数据清洗与列表、字典处理。
- 用户输入校验与异常处理。
- 文件读取、统计、写入与路径处理。
- 函数拆分、参数设计与返回值设计。
- 类与对象建模。
- 模块导入、包组织与复用。
- 正则表达式提取、校验与替换。
- 网络请求结果解析与响应数据整理。
- 综合项目中的小功能实现，例如订单统计、日志分析、配置解析、表单处理、客户信息管理。
