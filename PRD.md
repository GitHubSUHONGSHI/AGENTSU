# Python 课程学习站 PRD

## 1. 产品概述

### 1.1 产品名称

Python 课程学习站

### 1.2 产品形态

面向 Python 初学者的教学仪表盘网站。

### 1.3 产品背景

当前课程内容来源于 `Python1.0.docx`，文档包含从计算机基础、Python 快速入门、基础语法、容器、函数、文件操作、面向对象，到网络编程、正则表达式和综合案例的完整学习路径。

为了让课程内容更适合在线学习和后续迭代，需要将原始 docx 内容结构化为一个可浏览、可搜索、可追踪学习进度的前端网站。

### 1.4 目标用户

- Python 零基础或初学者
- 课程学员
- 助教和讲师
- 需要快速回顾 Python 基础知识的学习者

### 1.5 核心价值

- 将文档课程内容转为清晰的知识模块地图。
- 支持按章节浏览 Python 学习路径。
- 支持关键词搜索模块、章节和知识点。
- 支持本地学习进度记录。
- 为后续完整正文、代码示例、练习题和后台管理能力预留迭代空间。

## 2. 内容范围

### 2.1 首版内容范围

首版只做模块摘要和知识结构展示，不完整搬运 docx 正文、图片、表格和代码块。

首版需要展示：

- 17 个一级知识模块
- 每个模块下的二级章节
- 每个章节下的三级知识点
- 模块摘要
- 难度、预计学习时间、知识点数量等辅助信息

### 2.2 知识模块

根据 `Python1.0.docx` 解析，首版包含以下 17 个一级模块：

1. 概述
2. 快速入门
3. 基础知识
4. 流程控制语句
5. 容器数据类型
6. 函数
7. 文件操作
8. 面向对象之类和对象
9. 面向对象之三大特性
10. 面向对象案例：愤怒的小鸟
11. 错误和异常
12. 模块与包
13. Python 高级语法
14. 进程与线程
15. 网络编程
16. 正则表达式
17. 综合案例：客户信息管理系统

## 3. V1 功能需求

### 3.1 课程目录树

用户可以通过左侧目录树查看课程结构。

要求：

- 展示 17 个一级模块。
- 支持模块展开和折叠。
- 展示模块下的二级章节。
- 点击模块后，主内容区切换到对应模块详情。
- 当前选中模块需要有明确视觉状态。

### 3.2 模块详情展示

用户点击课程模块后，可以查看该模块的核心信息。

要求：

- 展示模块标题。
- 展示模块摘要。
- 展示难度标签。
- 展示预计学习时间。
- 展示章节数量和知识点数量。
- 展示学习完成状态。

### 3.3 章节和知识点列表

用户可以查看当前模块下的章节和知识点。

要求：

- 按二级章节分组展示。
- 每个章节展示三级知识点列表。
- 支持清晰的层级和阅读节奏。
- 不在首版展示完整正文。

### 3.4 关键词搜索

用户可以通过搜索框快速定位课程内容。

要求：

- 支持搜索一级模块名称。
- 支持搜索二级章节名称。
- 支持搜索三级知识点名称。
- 搜索结果需要可点击跳转到对应模块。
- 空搜索状态恢复默认课程结构。

### 3.5 学习进度标记

用户可以标记模块学习完成状态。

要求：

- 支持对一级模块标记已完成或未完成。
- 页面顶部展示整体学习进度。
- 模块列表中展示每个模块的完成状态。
- 进度计算基于已完成模块数和总模块数。

### 3.6 本地持久化

学习进度需要保存在浏览器本地。

要求：

- 使用 `localStorage` 保存已完成模块 ID。
- 页面刷新后学习进度仍然保留。
- 本地存储异常时不影响页面基础浏览能力。

### 3.7 响应式布局

网站需要适配桌面端和平板、移动端。

要求：

- 桌面端使用左侧目录 + 主内容区布局。
- 移动端课程目录使用抽屉或顶部入口展示。
- 375px、768px、1024px、1440px 视口下无横向滚动。
- 内容不重叠，按钮和文本不溢出。

## 4. 非功能需求

### 4.1 技术栈

- Vue 3
- Vite
- TypeScript
- Element Plus

### 4.2 Vue 开发规范

- 使用 `script setup lang="ts"`。
- 使用 Composition API。
- 明确声明 props 和 emits 类型。
- 派生状态使用 `computed`。
- 可复用逻辑放入 composables。
- 避免隐式 `any` 和 `as any`。
- 避免生产代码残留 `console.log`。

### 4.3 组件式开发

项目必须采用组件式开发。

推荐拆分：

- `components/course-sidebar.vue`
- `components/course-header.vue`
- `components/module-overview.vue`
- `components/section-list.vue`
- `composables/use-course-progress.ts`
- `composables/use-course-search.ts`
- `data/python-course.ts`
- `types/course.ts`

约束：

- 单个源码文件不得超过 300 行。
- 组件职责单一。
- 复杂逻辑拆分到 composables 或工具函数。
- 课程静态数据与 UI 组件分离。

### 4.4 Element Plus 使用规范

优先使用 Element Plus 提供的基础组件，避免重复造轮子。

推荐组件：

- `ElContainer`、`ElAside`、`ElHeader`、`ElMain`：整体布局
- `ElMenu` 或 `ElTree`：课程目录
- `ElInput`：关键词搜索
- `ElProgress`：学习进度
- `ElCard`：模块摘要和章节块
- `ElTag`：难度、时长、数量等元信息
- `ElButton`、`ElCheckbox`：学习状态操作
- `ElDrawer`：移动端目录

### 4.5 可访问性

- 交互元素必须可通过键盘访问。
- 点击元素优先使用 `button` 或 Element Plus 交互组件。
- 展开和折叠状态需要与 ARIA 状态同步。
- 页面标题层级保持清晰。
- 文本颜色和背景需要具备足够对比度。
- 不使用颜色作为唯一状态表达。

### 4.6 UI/UX 方向

产品采用教育类教学仪表盘风格。

设计方向：

- 清晰、稳定、适合长期学习。
- 左侧课程树负责导航。
- 主内容区负责模块理解。
- 顶部负责搜索和进度概览。
- 使用 teal 作为进度主色，orange 作为强调色。
- 在 Element Plus 基础上做适度主题覆盖，避免默认后台模板感。

## 6. 技能与审查流程

### 6.1 ui-ux-pro-max

用途：

- 生成 UI/UX 设计系统参考。
- 辅助确定教育 dashboard 的布局、颜色、字体和可访问性约束。
- 检查响应式、键盘导航、对比度和视觉层级。

### 6.2 frontend-code-review

用途：

- 审查 Vue、TypeScript、JavaScript 和前端组件代码。
- 检查 Vue SFC 结构是否清晰。
- 检查组件职责、TypeScript 类型、可访问性和交互细节。
- 检查是否符合 Element Plus 和 Vue3 的主流开发习惯。

### 6.3 code-reviewer

用途：

- 做通用代码质量复核。
- 检查 XSS、硬编码敏感信息、不安全数据处理。
- 检查冗余计算、内存泄漏、缺少错误处理等问题。
- 检查命名、DRY、可维护性和最佳实践。
- 输出整体代码质量评分。

## 7. 版本路线

### 7.1 V1：课程地图与模块摘要

目标：

- 完成课程学习站基础框架。
- 展示 17 个模块、章节和知识点。

目标：

## 8. 验收标准

### 8.1 功能验收

- 17 个一级模块完整展示。

- 不提供登录和注册。

- 不处理 docx 内图片和表格。
- 不提供在线 Python 代码执行能力。
- 不提供后台课程管理能力。
